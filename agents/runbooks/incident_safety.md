# Runbook: Safety Incident (kritischer Inhalt)

## Ziel
Schnell riskante Templates/Prompts/Modelle deaktivieren, Nutzer schützen, und sauber dokumentieren.

## Ablauf (Kurz)
- [ ] Incident eröffnen (Ticket/Doc) + Owner festlegen
- [ ] Sofortmaßnahme: betroffene Prompt-Version/Template/Model deaktivieren
- [ ] Logs prüfen (datensparsam) + Umfang abschätzen
- [ ] Hotfix: Guardrails/Policy/Evals erweitern
- [ ] Postmortem: Ursache, Maßnahmen, Regression-Evals
