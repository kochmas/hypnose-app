# Spikes / Experimente

Hier dokumentieren wir kleine, reproduzierbare “Spikes”, um technische Entscheidungen (Queue/Worker/Hosting/Auth etc.) praktisch zu testen.

Konvention:
- Eine Datei pro Spike: `SPIKE-DEC-XXX-kurzer-name.md`
- Kurz halten: Hypothese → Test → Ergebnis → Empfehlung
- Nach Abschluss: die zugehörige DEC in `planung/entscheidungen.md` aktualisieren und ggf. ins Archiv verschieben.
